# 測試需求

**English:** [`rules/common/testing.md`](../../../rules/common/testing.md)

## 最低測試覆蓋率：80%

測試類型（**皆必要**）：

1. **單元測試** — 單一函式、工具、元件。
2. **整合測試** — API 端點、資料庫操作等。
3. **E2E 測試** — 關鍵使用者流程（依語言／框架選用工具）。

## 測試驅動開發（TDD）

**必須**遵守的流程：

1. 先寫測試（**RED**）。
2. 執行測試 — 應**失敗**。
3. 寫最少實作使測試通過（**GREEN**）。
4. 再執行測試 — 應**通過**。
5. 重構（**IMPROVE**）。
6. 確認覆蓋率 **80%+**。

## 測試失敗時

1. 使用 **tdd-guide** agent。
2. 檢查測試是否彼此隔離。
3. 確認 mock 是否正確。
4. 優先修**實作**，而非測試（除非測試本身錯了）。

## Agent 協助

- **tdd-guide** — 新功能、修 bug、重構時**主動**使用；強制先測試後實作。
